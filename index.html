سياسة الخصوصية لتطبيق Kees

```

آخر تحديث: 15 يونيو 2026

توضح هذه السياسة كيفية جمع واستخدام وحماية البيانات عند استخدام تطبيق Kees. باستخدامك للتطبيق، فإنك توافق على ممارسات الخصوصية الموضحة في هذه الصفحة.

1. البيانات التي نقوم بجمعها

قد يجمع تطبيق Kees بعض البيانات اللازمة لتشغيل الخدمة وتحسين تجربة المستخدم، وتشمل:

  • الاسم ورقم الهاتف وبيانات الحساب الأساسية.
  • البريد الإلكتروني إذا تم استخدامه أو توفيره داخل الحساب.
  • المنطقة المرتبطة بالمستخدم أو المتجر أو السائق.
  • بيانات الطلبات، السلة، المنتجات، المتاجر، العروض، والكوبونات.
  • موقع التوصيل أو موقع السائق أثناء وجود طلب نشط أو عند استخدام خدمات الموقع داخل التطبيق.
  • الصور أو الملفات التي يتم رفعها، مثل صور المنتجات أو المتاجر أو العروض أو الملفات اللازمة لتشغيل الخدمة.
  • رموز الإشعارات مثل FCM Token لإرسال التنبيهات المهمة المتعلقة بالطلبات والحساب.
  • سجلات تشغيلية وإدارية تساعد في حماية الحسابات، حل المشاكل، وتحسين الخدمة.

2. كيفية استخدام البيانات

نستخدم البيانات للأغراض التالية:

  • إنشاء الحساب وتسجيل الدخول وإدارة المستخدمين.
  • عرض المتاجر والمنتجات والعروض حسب المنطقة.
  • إنشاء الطلبات ومتابعتها وإرسال إشعارات حالتها.
  • تمكين السائق من استلام الطلبات وتحديث حالة التوصيل.
  • تمكين المتاجر من إدارة المنتجات والطلبات والعروض.
  • إدارة الكوبونات والمستحقات والسجلات المالية الداخلية.
  • إرسال إشعارات مهمة مثل الطلبات الجديدة، حالة الطلب، وصول السائق، العروض، والتنبيهات الإدارية.
  • تحسين أداء التطبيق، معالجة الأخطاء، ومنع إساءة الاستخدام أو الاحتيال.

3. بيانات الموقع

يستخدم التطبيق بيانات الموقع فقط عند الحاجة لتشغيل وظائف أساسية مثل تحديد منطقة المستخدم، اختيار موقع التوصيل، تتبع الطلب، أو تمكين السائق من تحديث موقعه أثناء وجود طلب نشط.

لا يتم استخدام الموقع لتتبع المستخدم بشكل غير ضروري، ولا يتم استخدامه لأغراض بيع البيانات. يمكن للمستخدم التحكم بصلاحيات الموقع من إعدادات الجهاز.

4. الصور والملفات

قد يسمح التطبيق برفع صور المنتجات، المتاجر، العروض، أو صور أخرى مرتبطة بالخدمة. يتم استخدام هذه الصور فقط لتشغيل وظائف التطبيق وعرض المحتوى داخل Kees.

5. الإشعارات

قد يرسل التطبيق إشعارات للمستخدمين أو المتاجر أو السائقين أو الإدارة، مثل:

  • تحديثات حالة الطلب.
  • تنبيهات وصول السائق أو وجوده بالخارج.
  • العروض الجديدة والتنبيهات المهمة.
  • إشعارات إدارية وتشغيلية مرتبطة بالحساب أو الطلبات.

يمكن للمستخدم التحكم بالإشعارات من إعدادات الجهاز، وقد يوفر التطبيق إعدادات داخلية لبعض أنواع الإشعارات.

6. مشاركة البيانات مع أطراف خارجية

يستخدم التطبيق خدمات خارجية ضرورية لتشغيله، مثل Firebase وFirestore وCloud Functions وFirebase Storage وFirebase Cloud Messaging، وقد يتم استخدام خدمات خرائط أو موقع عند الحاجة. تتم مشاركة البيانات مع هذه الخدمات فقط بالقدر اللازم لتشغيل التطبيق.

لا نقوم ببيع بيانات المستخدمين الشخصية لأي طرف ثالث.

7. السجلات المالية والطلبات

يحتفظ التطبيق بسجلات مرتبطة بالطلبات، مستحقات المتاجر، مستحقات السائقين، الكوبونات، العروض، والعمليات الإدارية. تستخدم هذه البيانات لأغراض التشغيل، المحاسبة، الدعم، الأمان، وحل النزاعات.

8. حماية البيانات

نستخدم إجراءات تقنية وتنظيمية للمساعدة في حماية البيانات من الوصول غير المصرح به أو التعديل أو الحذف غير المشروع. ورغم ذلك، لا توجد طريقة نقل أو تخزين إلكترونية آمنة بنسبة 100%.

9. الاحتفاظ بالبيانات وحذف الحساب

نحتفظ بالبيانات طالما كانت ضرورية لتشغيل الخدمة، تنفيذ الطلبات، الدعم، المتابعة، الأمان، أو المتطلبات القانونية والمحاسبية.

يمكن للمستخدم طلب حذف حسابه وبياناته من داخل التطبيق من خلال:

حسابي ← إعدادات الحساب ← حذف الحساب

بعد تقديم طلب الحذف، يتم وضع الحساب في حالة حذف معلّق لمدة 14 يومًا. خلال هذه الفترة يمكن للمستخدم تسجيل الدخول والاطلاع على حالة الطلب والمدة المتبقية، ويمكنه إلغاء طلب الحذف قبل التنفيذ النهائي.

بعد مرور 3 أيام من طلب الحذف، قد تبدأ إجراءات تجهيز الحساب للحذف. وبعد انتهاء مهلة 14 يومًا، يتم حذف الحساب والبيانات الشخصية المرتبطة به قدر الإمكان.

قد نحتفظ ببعض السجلات الضرورية مثل سجلات الطلبات أو المعاملات أو السجلات المحاسبية عند الحاجة لأغراض قانونية أو تشغيلية أو لمنع الاحتيال، مع تقليل البيانات الشخصية المباشرة قدر الإمكان.
```
